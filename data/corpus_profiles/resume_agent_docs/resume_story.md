# Resume Story For Agent Internship

The project is best presented as a trustworthy AI application rather than a simple RAG demo.
The interview story should start from the user question, then explain Planner, DAG execution,
SQLite memory, vector recall, TextRank compression, Verifier, Red-Blue repair, and evaluation.

Recruiters should see a runnable artifact. A local FastAPI demo can load the final evidence pack,
start a mock offline run, and visualize report claims, citations, evidence chunks, verification
traces, and repair actions.

Metrics must be framed carefully. Offline mock benchmark results can compare internal system
configurations, but real DeepSeek provider smoke should not be mixed into formal benchmark scores.
